# 智能对话系统中期答辩PPT

## 1. 项目背景与意义

### 1.1 项目背景
- 随着人工智能技术的快速发展，智能对话系统在日常生活和工作中的应用越来越广泛
- 传统的对话系统往往存在响应速度慢、用户体验差等问题
- 本项目旨在开发一个响应迅速、用户友好的智能对话系统

### 1.2 项目意义
- 提供便捷的智能对话服务，满足用户的信息需求
- 探索前端与后端的协同开发模式，提升系统性能
- 为类似项目的开发提供参考和借鉴

## 2. 技术栈选择

### 2.1 前端技术
- **React**：用于构建用户界面的JavaScript库，具有组件化、声明式的特点
- **Vite**：现代化的前端构建工具，提供快速的开发体验和高效的构建过程
- **CSS**：用于样式设计，实现美观的用户界面

### 2.2 后端技术
- **Node.js**：基于Chrome V8引擎的JavaScript运行环境，用于构建高性能的后端服务
- **Express**：轻量级的Web应用框架，提供简洁的API设计
- **JSON**：用于数据存储，简单直观，适合小型应用

### 2.3 第三方API
- **Dify API**：提供智能对话能力，支持流式响应

## 3. 系统架构

### 3.1 系统架构图
```
+-------------------+    +-------------------+    +-------------------+
|                   |    |                   |    |                   |
|   前端界面        |<-->|   后端服务        |<-->|   智能对话API     |
|   (React + Vite)  |    |   (Node.js + Express) |   (Dify API)      |
|                   |    |                   |    |                   |
+-------------------+    +-------------------+    +-------------------+
        ^                         |
        |                         |
        +-------------------------+
                  |
                  v
        +-------------------+
        |                   |
        |   本地存储        |
        |   (JSON文件)      |
        |                   |
        +-------------------+
```

### 3.2 核心流程
1. 用户在前端界面输入问题
2. 前端将问题发送到后端服务
3. 后端服务调用智能对话API获取回答
4. 智能对话API返回流式响应
5. 后端服务将回答实时传递给前端
6. 前端实时显示回答内容
7. 后端服务将对话记录保存到本地存储

## 4. 核心功能实现

### 4.1 对话管理
- **创建新对话**：用户可以创建新的对话，系统会生成唯一的对话ID
- **加载历史对话**：系统会自动加载历史对话列表，用户可以点击切换
- **删除对话**：用户可以删除不需要的对话记录

### 4.2 智能对话
- **实时响应**：支持流式响应，用户可以看到AI的回答过程
- **错误处理**：当API调用失败时，系统会给出友好的错误提示
- **对话保存**：系统会自动保存对话记录，确保数据不丢失

### 4.3 用户界面
- **响应式设计**：适配不同屏幕尺寸，提供良好的用户体验
- **美观的界面**：采用现代化的设计风格，界面简洁美观
- **流畅的交互**：优化了对话切换逻辑，确保操作流畅
